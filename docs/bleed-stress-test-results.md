# Bleed stress test results (MIC-316)

Automated coverage lives in `UnitTests/tst_bleedstresstest.cpp`. Fixtures are generated by `frisket-preflight/tools/generate_fixtures.py` and listed in `frisket-preflight/testdata/fixtures/manifest.json`.

Profile: `frisket-preflight/examples/profile-tiered-bleed-raster.json` (9 pt bleed, raster content-bleed confirmation).

## Fixtures

| Fixture | Scenario |
|---------|----------|
| `ai-art-missing-bleed.pdf` | CMYK raster with geometric edge lines flush to trim; all boxes equal (zero bleed margin). |
| `ai-art-partial-bleed.pdf` | Adequate bleed boxes on left/bottom only; right/top still short. |
| `ai-art-hard-corners.pdf` | High-contrast checker blocks in all four corners at trim (mirror seam stress). |
| `ai-art-raster-trim-edge.pdf` | Photo-like RGB gradient raster touching trim on all sides. |

## Mode guidance (V1 manual review)

| Pattern | Recommended mode | Notes |
|---------|------------------|-------|
| Missing bleed, uniform edge artwork | **Mirror** | Best default when edges are continuous tone or soft gradients. |
| Partial bleed / mixed edges | **Mirror** | Fills all short sides in one pass; verify right/top in overlay. |
| Hard corners / checker seams | **Pixel repeat** or **Stretch** | Mirror can show a visible seam on high-contrast corners; pixel-repeat is often less obvious on 1 px strips. |
| Photo raster at trim | **Mirror** | Preserves color continuity better than stretch; watch for duplicated detail on busy texture. |

## Known limitations

- Mirror mode can produce visible seams on high-contrast geometric corners (`ai-art-hard-corners`).
- Stretch mode can distort fine detail or text near the trim line.
- Pixel repeat is fast but can tile obvious patterns on wide bleed strips.
- RGB artwork in `ai-art-raster-trim-edge` is repaired for bleed geometry only; color-mode policy is outside this stress profile.

## Running tests

```bash
# Generate fixtures first (requires reportlab, pillow, pikepdf)
python3 frisket-preflight/tools/generate_fixtures.py

# After building PdfTool + UnitTestsBleedStress
ctest -R UnitTestsBleedStress --output-on-failure
```
